# TASKS_CLAUDE_CODE.md — Ordre d’exécution Claude Code

## Mode de travail

Ne code pas tout d’un coup.

Procède par étapes. Après chaque étape, produire un récapitulatif et attendre validation humaine.

## Étape 1 — Analyse sans code

Lire :

- `CLAUDE.md`
- `docs/BRIEF_CONSOLIDE_SITE_PUBLIC_V0.md`
- `assets/maquette_home_reference.png`
- `docs/CHECKLIST_RECETTE.md`

Produire uniquement :

1. le périmètre du site public V0 ;
2. les pages à créer ;
3. les interdictions ;
4. les points techniques à arbitrer ;
5. le plan d’implémentation proposé.

Ne pas coder avant validation humaine.

## Étape 2 — Structure du site

Créer la structure des pages :

- `/`
- `/merci`
- `/contact`
- `/installateurs`
- `/mentions-legales`
- `/confidentialite`

Créer les éléments globaux :

- Header ;
- Footer localisé ;
- Layout responsive ;
- Ancres de navigation.

Stop après cette étape pour validation.

## Étape 3 — Home page

Coder la home dans l’ordre verrouillé :

1. Header
2. Hero avec formulaire étape 1 visible
3. Bénéfices accès autonome
4. Problème des clés à distance
5. Solutions
6. Qualification du besoin
7. Cibles
8. Comment ça marche
9. Réassurance
10. FAQ
11. CTA final
12. Footer localisé

Utiliser le wording exact du brief.

Stop après cette étape pour validation.

## Étape 4 — Formulaire

Coder le formulaire 3 étapes.

Respecter :

- étape 1 visible en hero ;
- étapes 2 et 3 fermées au départ ;
- règles réponse unique / multi-réponses ;
- options “Je ne sais pas” exclusives ;
- upload photo facultatif ;
- créneaux de rappel ;
- mention obligatoire sur les créneaux ;
- aucun rendez-vous confirmé ;
- aucun devis automatique.

Stop après cette étape pour validation.
