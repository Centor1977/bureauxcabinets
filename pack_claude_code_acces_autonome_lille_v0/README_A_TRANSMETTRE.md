# Pack développeur — Accès Autonome Lille V0

Ce dossier contient les documents à transmettre au développeur qui utilisera Claude Code pour construire le site public V0.

## Règle principale

Le développeur et Claude Code doivent exécuter le brief. Ils ne doivent pas réinterpréter la promesse, la direction artistique, le wording, l’ordre des sections, le formulaire ou le périmètre.

## Ordre de lecture recommandé

1. `CLAUDE.md`
2. `TASKS_CLAUDE_CODE.md`
3. `docs/BRIEF_CONSOLIDE_SITE_PUBLIC_V0.md`
4. `assets/maquette_home_reference.png`
5. `docs/CHECKLIST_RECETTE.md`

Les autres fichiers dans `docs/` détaillent chaque partie du site.

## Important

Ce pack verrouille le site public à développer : pages, structure, wording, parcours visible, formulaire, direction artistique et critères de recette.

Ce pack ne verrouille pas les choix techniques : stack, base de données, stockage des photos, hébergement, emails, tracking, CRM, automatisations. Ces sujets sont à arbitrer séparément avec le développeur.
